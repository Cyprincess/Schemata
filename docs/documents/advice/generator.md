# Advice Generator

`Schemata.Advice.Generator` is a Roslyn incremental source generator that eliminates boilerplate `RunAsync` extension methods. For every interface in the compilation whose base list contains `IAdvisor`, the generator emits a `RunAsync` extension method on `AdvicePipeline<TAdvisor>` that delegates to the correct `AdviceRunner<...>` overload. Adding a new advisor interface requires only the interface declaration; the generator handles the rest.

## Where the code lives

| Package | Key files |
| --- | --- |
| `Schemata.Advice.Generator` | `AdvicePipelineGenerator.cs` |
| `Schemata.Advice.Generator` | `AdvisorInterfaceInfo.cs` |

The generator targets `netstandard2.0` and references `Microsoft.CodeAnalysis.CSharp` as `PrivateAssets="all"`. It is consumed as an `Analyzer` reference by every `src/` project via `Directory.Build.props`.

## Activation guard

The generator activates only when `Schemata.Advice.AdvicePipeline\`1` is present in the compilation:

```csharp
var hasInfrastructure = context.CompilationProvider.Select(
    static (compilation, _) =>
        compilation.GetTypeByMetadataName("Schemata.Advice.AdvicePipeline`1") is not null);
```

If the runtime package is absent, no source is emitted. This prevents the generator from producing uncompilable code in projects that reference only `Schemata.Abstractions`.

## Candidate detection

The generator uses a fast syntactic filter before semantic analysis:

```csharp
private static bool IsAdvisorCandidate(SyntaxNode node) {
    if (node is not InterfaceDeclarationSyntax iface) return false;
    if (iface.BaseList is null) return false;

    foreach (var baseType in iface.BaseList.Types) {
        if (baseType.Type.ToString().Contains("IAdvisor")) return true;
    }

    return false;
}
```

Any interface whose base list contains the text `"IAdvisor"` is a candidate. This is a text-based check on the syntax tree, not a semantic check, so it runs before the semantic model is consulted.

## Semantic analysis

For each candidate, `GetAdvisorInfo` resolves the symbol and walks `AllInterfaces` and `Interfaces` to find the first `IAdvisor` generic base:

1. Finds the `IAdvisor<T1, ..., TN>` base interface and extracts its type arguments.
2. Collects the interface's own type parameters and their constraints.
3. Builds the `RunAsync` method parameter list: `AdviceContext ctx`, then one parameter per type argument (`a1`, `a2`, ...), then `CancellationToken ct = default`.
4. Builds the `AdviceRunner<...>` type argument list: the advisor interface type, then each type argument.

The result is an `AdvisorInterfaceInfo` record holding all the strings needed to emit the extension method.

## Code emission

For each `AdvisorInterfaceInfo`, the generator emits a partial class `AdvicePipelineExtensions` in the `Schemata.Advice` namespace:

```csharp
// <auto-generated/>
#nullable enable

namespace Schemata.Advice;

public static partial class AdvicePipelineExtensions
{
    public static global::System.Threading.Tasks.Task<global::Schemata.Abstractions.Advisors.AdviseResult>
        RunAsync<TEntity>(
            this global::Schemata.Advice.AdvicePipeline<
                global::Schemata.Entity.Repository.Advisors.IRepositoryAddAdvisor<TEntity>> _,
            global::Schemata.Abstractions.Advisors.AdviceContext ctx,
            global::Schemata.Entity.Repository.IRepository<TEntity> a1,
            TEntity a2,
            global::System.Threading.CancellationToken ct = default)
        where TEntity : class
        => global::Schemata.Advice.AdviceRunner<
               global::Schemata.Entity.Repository.Advisors.IRepositoryAddAdvisor<TEntity>,
               global::Schemata.Entity.Repository.IRepository<TEntity>,
               TEntity>
           .RunAsync(ctx, a1, a2, ct);
}
```

All type references use `global::` fully-qualified names to avoid namespace conflicts.

## Hint name generation

Each generated file is registered with `spc.AddSource(hintName, source)`. The hint name is derived from `AdvisorInterfaceInfo.InterfaceMinimalName` by replacing characters that are invalid in file names:

```csharp
var hintName = info.InterfaceMinimalName
    .Replace("global::", "")
    .Replace("<", "_")
    .Replace(">", "_")
    .Replace(", ", "_")
    .Replace(".", "_");
spc.AddSource($"{hintName}.g.cs", sb.ToString());
```

`InterfaceMinimalName` is the interface's simple name plus a `_TypeParam` suffix for generic interfaces (e.g. `IRepositoryAddAdvisor_TEntity`).

### Hint-name collision risk

Two interfaces with the same simple name in different namespaces produce the same hint name. For example:

```csharp
namespace Foo { public interface IMyAdvisor<T> : IAdvisor<T> { } }
namespace Bar { public interface IMyAdvisor<T> : IAdvisor<T> { } }
```

Both produce hint name `IMyAdvisor_T.g.cs`. Roslyn treats duplicate hint names as a conflict and may silently drop one of the generated files. To avoid this, use unique simple names across all advisor interfaces in a compilation, or include a namespace-derived prefix in the interface name.

## Type parameter forwarding

When the advisor interface itself has type parameters (e.g. `IRepositoryAddAdvisor<TEntity>`), the generator forwards them to the emitted `RunAsync` method and to the `AdviceRunner<...>` type arguments. Constraints declared on the interface's type parameters are reproduced on the emitted method.

The generator resolves type arguments that are the interface's own type parameters by name (not by fully-qualified display), so the emitted code uses `TEntity` rather than `global::System.Object` in the runner type arguments.

## Extension points

- Define a new advisor interface that extends `IAdvisor<T1>` (or any higher arity up to 16). The generator emits the `RunAsync` extension automatically on the next build.
- The generator does not emit anything for non-interface types, for interfaces without an `IAdvisor` base, or for interfaces in projects where `Schemata.Advice.AdvicePipeline\`1` is absent.
- To suppress generation for a specific project, set `<SchemataSkipGenerators>true</SchemataSkipGenerators>` in the project file. This removes the `Analyzer` reference injected by `Directory.Build.props`.

## Design motivation

The incremental generator pattern (`IIncrementalGenerator`) means the generator only re-runs for interfaces that changed since the last build. The syntactic pre-filter (`IsAdvisorCandidate`) keeps the semantic model queries to a minimum. The activation guard prevents the generator from emitting code that references `AdvicePipeline<T>` in projects that don't have the runtime package.

## Caveats

- The candidate detection uses a text-based check (`Contains("IAdvisor")`). An interface with `IAdvisor` anywhere in its base list text — even in a comment-like position — will be treated as a candidate and subjected to semantic analysis. This is harmless but slightly wasteful.
- The generator walks `AllInterfaces` first, then `Interfaces` (direct bases). The last match in `Interfaces` wins. If an advisor interface has multiple `IAdvisor` bases (unusual), only the last direct `IAdvisor` base is used.
- Hint-name collisions (same simple name in different namespaces) can silently drop one generated file. Use unique simple names across all advisor interfaces in a compilation.
- The generator emits into the `Schemata.Advice` namespace regardless of where the advisor interface is declared. All generated files contribute to the same `AdvicePipelineExtensions` partial class.
